Methodology

This section explains the research logic behind the project: how violent events are structured, classified and translated into a visual interface.

View data structure Read the analysis

Event-based approach.

The project uses an event-data logic inspired by conflict research practices. Each observation is treated as a discrete violent incident and organised through standard fields such as date, location, coordinates, actor or dyad, type of violence, fatalities, civilian deaths and source information.

01

Observation

Each row corresponds to a specific violent event rather than a broad narrative category.

02

Classification

Events are coded according to violence type, dyad, civilian targeting and available source detail.

03

Spatialisation

Coordinates allow the dataset to be explored through markers, clusters and spatial concentration layers.

Coding principles.

Violence is coded conservatively when perpetrator identity, affiliation or motive remains uncertain.

Where civilians are targeted by armed actors, the event is treated as one-sided violence unless reliable information indicates a direct clash between organised groups.

Ambiguous cases require source comparison, contextual reading and, when necessary, later revision as new information appears.

Spatial precision is treated as an analytical variable: city-level, neighbourhood-level and street-level references are not interpreted as equivalent.

Why visualise the data?

The map is not meant to replace qualitative analysis. It works as an exploratory tool that helps identify concentrations, recurring locations and spatial patterns that can then be interpreted through fieldwork, literature and source-based investigation.
